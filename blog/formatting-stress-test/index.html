paragraphs and inline formatting

this is a normal paragraph. the body text should be comfortable to read at the blog’s 600px max-width — roughly 60-70 characters per line, which is the sweet spot for readability. the color should be dark enough for sustained reading without being harsh.

bold text draws the eye to key terms. italic text adds emphasis without weight. bold italic combines both. sometimes you need inline code in a sentence — like when referencing a command (extro-blog publish) or a file path (cmo/workspace/blog/). inline code should be visually distinct but not jarring.

here’s a paragraph with a link to solpbc.org and another to the blog index. links should be clearly identifiable (orange accent) without overwhelming the surrounding text. on hover, they should underline.

a short paragraph for rhythm.

and then a longer one that explores an idea in more depth, because real blog posts have varied paragraph lengths and the spacing between them needs to work for both. the eye should flow naturally from one paragraph to the next without feeling cramped or disconnected. this paragraph is deliberately longer to test how the line-height and paragraph spacing hold up over multiple lines of continuous text that a reader would actually sit with.

unordered lists

things sol pbc will never do:

  • sell user data in any form — anonymized, aggregated, or otherwise
  • accept an acquisition offer — the articles of incorporation prohibit it
  • hire employees who could access user data — the founder is the only person
  • install tracking or analytics on any property
  • use third-party ad networks or behavioral targeting
